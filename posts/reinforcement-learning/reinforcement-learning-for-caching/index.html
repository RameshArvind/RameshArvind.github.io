Reinforcement Learning for Caching

Databases are ubiquitous in the world of internet companies. Almost every company today has a system of records for their users, content, transactions and a variety of other forms of data. Processing large amounts of data efficiently has been a key component of database research for many decades by optimizing different database components.

One such component is the cache. It is used to reduce the amount of disk reads / writes to optimize latency. A widely used caching algorithm is the Least Recently Used (LRU) algorithm which is simple but efficient. With the increasing amounts of data, research has been focused on implementing an adaptive caching strategy that learns from past database accesses. This area of research is at the intersection of Systems and Machine Learning.

In this special problem we wish to explore various facets of the caching algorithm such as

Prefetching

While processing large workloads, access different memory segments each time is a big bottleneck to database performance. Strategies to predict which memory block will be accessed next is a big improvement

Cache Admission

Once a memory block has been accessed, determining if it needs to be cached or not helps improve cache efficiency. Pre-emptively caching a short lived object will not only reduce cache storage efficiency, but it also increases number of cache misses due to eviction of an existing object.

Cache Eviction

Managing the cache is also important. Ensuring that the cache consists of only “hot” objects is of vital importance to maximize cache hits. Removing “cold” entries allows the next “hot” objects to be cached.

TTL Prediction

This is a subset of the cache eviction policy, in that predicting how long an object will remain “hot” allows the cache to clean up the cache without having to make a decision on every cache admission. With a TTL attached to every object, the cache can asynchronously evict objects whose TTL has passed. The TTL is also an indicator of the value of the object with respect to cache.

Capacity Aware Caching

With homogeneous object sizes, the decision to evict/admit an object is purely a function of its “hotness”. In a heterogeneous setting since the cache is usually a fixed size, one must also consider the size of objects being stored in addition to the “hotness” factor. A large object which is also hot is relatively less efficient than a smaller object which is equally “hot”. Considering the size of objects in the caching decision improves the efficiency of the cache in a constrained setting.

There has been existing work done on applying reinforcement learning for learning cache eviction policy using Deep Q Networks and using supervised approaches for Cache admission / prefetching tasks.

There have also been approaches which tackle all the various facets as one multi-agent reinforcement learning problem and benchmarked it against the Yahoo Cloud Serving benchmark.

This task has various applications in web, filesystem and virtually any system that utilizes a caching policy with actively changing trends of access patterns.

Various Tasks outlined for study:

  • Understand the existing implementations of cache replacement policies
  • Formulate the cache eviction problem and benchmarking strategies
  • Implementation of reinforcement learning policy for caching
  • Evaluating our learned policy with existing policies

We conduct this research with the aim of publishing it in a peer reviewed journal.
